03 · 模型之战 · 原理一：AI 是什么

大模型的本质：一个学会了"接话"的超大函数

它不查答案，它算概率——你看到的所有"智能"，都来自一次次对下一个词的预测。

参数 = 万亿个旋钮

模型是一张巨大的数学网络，"参数"是网络里可调的旋钮，前沿模型有上万亿个。所谓训练，就是把这些旋钮拧到正确的位置。

Token = 文字的积木

模型把文字切成小块处理（"人工智能"≈ 2 个 token）。API 按 token 计费——后面价格页的"每百万 token"就是这么来的。

涌现 = 量变到质变

规模定律（Scaling Law）：参数、数据、算力一起放大，理解、推理、创造这些能力会"自己长出来"——这就是全世界疯抢算力的原因。

一句话总结：AI 不是被"编程"出来的，是被"喂"出来的——它从海量文本中学到了语言乃至世界的规律。
03 · 模型之战 · 原理二：训练是怎么回事

"炼"一个大模型：先读完整个互联网，再教它听话

① 预训练

读遍数万亿 token 的文本（≈ 几百万个图书馆），在猜词游戏中学会语言与世界的规律。算力和钱主要烧在这一步。

② 后训练

人类示范 + 打分（RLHF 等），教它有用、诚实、守规矩——从"会接话"变成"会干活"。

③ 推理（使用）

你每一次提问都是一次"推理"，按 token 计费。训练一次，服务亿万人。

Claude Code 在终端里的工作循环

① 理解

自己读代码库、查文档，搞清现状。

② 计划

拆解任务，列出方案，重大操作先征求同意。

③ 执行

写代码、改文件、跑命令、装依赖，多步连续作业。

④ 验证

自己运行测试、检查报错、修到通过为止。

关键区别：出错时它不是把报错丢给你，而是读懂报错、改完再试——这正是"工具"与"同事"的分界线。

05 · Agent · 案例 · Claude Code 三个典型工作流（现场可演示）

一句话任务，它能走多远

案例一 · 从零建站

"帮我做一个班级活动报名网页，能填名字、选场次、导出名单。"

它会：搭页面 → 写表单逻辑 → 加数据存储 → 本地跑起来给你看。传统上要学几周 HTML/JS 才能做的事，20 分钟出可用版本。

案例二 · 接手陌生项目修 Bug

"这个开源项目跑起来报这个错，帮我修好。"（贴上报错）

它会：读报错 → 在几万行代码里定位根因 → 修改 → 跑测试确认。看懂别人的烂代码恰恰是 AI 最省你命的地方。

案例三 · 数据分析自动化

"这份问卷 Excel，帮我清洗数据、做交叉分析、生成图表报告。"

它会：写脚本清洗 → 统计分析 → 出图 → 生成一页网页报告。不用会 Python，但你要会判断结论对不对。

共同点：你的角色从"操作者"变成"提需求、做判断、验收结果的人"——这三件事，就是接下来练习要练的。

Hermes：会自我进化的智能体

它是什么 MIT 开源

Nous Research（开源模型 Hermes 系列同门）2026 年 2 月发布的通用个人 Agent，口号"The agent that grows with you"。部署在自己的服务器上，强调数据主权；一个网关接入 20+ 聊天平台（微信/钉钉/飞书/Telegram…），兼容 200+ 大模型；6 月刚推出对新手友好的桌面版。

05 · Agent

对普通人来说，这意味着什么

  • 编程门槛消失了，但工程判断没有。AI 替你写代码，不替你决定"该做什么、做成什么样、对不对"。
  • 被替代的不是"工程师"，是"不会用 AI 的人"。会拆解问题、会精确提需求、会验证结果，是新的核心竞争力。
  • 提问质量 = 产出质量。模糊的要求得到模糊的结果；说清目标、约束、验收标准，产出立刻不同。
  • 普通人几乎没有"经验劣势"。这套工作方式 2025 年才出现、2026 年才真正普及，你和从业十年的人站在几乎同一条起跑线上——这在历史上极少发生。

旧赛道的里程，带不进新赛道——两条前进的路，一样长。

实战练习

五道题，每道都按同一个框架推进：

目标

用一句话说清要做出什么

工具

选用哪个 AI / 怎么开工

交付物

做完后拿什么东西验收

复盘

哪句提示词最关键？AI 错在哪？

练习一 · 数据基建

让 Agent 搭一条"自动抓取 → 清洗 → 入库"的数据管道（行情 / 新闻 / 天气任选其一），能重复运行。
交付物：跑通的脚本 + 一份干净的数据文件。

练习二 · 投资分析

选一家今天讲过的 AI 公司（智谱 / MiniMax / 英伟达…），让 AI 联网核实数据后产出一页投资简报：业务、估值、风险。
交付物：一页可分享的网页或文档简报。

练习三 · 视频转写

把一段演讲或课程视频交给 AI：转文字稿 → 整理成带小标题、金句摘录的结构化笔记。
交付物：一份可直接发布的图文笔记。

练习四 · Hermes 工具

在自己电脑上部署 Hermes Agent，接入一个聊天软件，指挥它完成一个任务并沉淀为技能。
交付物：成功对话的截图 + 沉淀下来的技能文档。

练习五 · 自选题

用今天学到的任何工具，解决一个你自己学习 / 生活里的真实问题。唯一要求：开工前说清目标和验收标准，做完后复盘 AI 错在哪。

提示：练习不比谁做得快，比谁说得清、验得准——提问质量 = 产出质量。
