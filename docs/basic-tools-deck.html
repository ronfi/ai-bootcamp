Terminal：学会和电脑说话

图形界面是给"用户"的，终端是给"指挥官"的。GUI 里你只能做开发者预设的事，终端里你能说出任何指令——而且 AI 听得懂。

01 · Terminal

终端 = 文字对话框，Shell = 听懂人话的翻译官

终端（Terminal）

一个用打字代替点鼠标的窗口。你输入命令，它返回结果——就这么简单。

Shell（命令解释器）

终端里真正干活的程序：Linux 常用 bash，macOS 默认 zsh，Windows 是 PowerShell。它们像不同方言，核心语法相通。

为什么值得学

① 精确——一条命令就是一句无歧义的指令；② 可复制——发给别人照跑就行；③ 服务器没有图形界面；④ 它是 AI 智能体的母语。

$ echo "你好，终端"
你好，终端
$ ls
training  stock-report  notes.txt
$ # 井号后面是注释，解释这条命令在干嘛
$ ▌
心理建设：终端不会因为你打错命令就坏掉——大多数错误只是提示"找不到命令"。大胆敲。
01 · Terminal · 各平台怎么打开

三大平台的入口，Windows 请直接装 WSL

macOS 自带

⌘ + 空格 搜 "Terminal"（终端）回车即得，默认 shell 是 zsh。进阶可装 iTerm2。

Linux 自带

桌面版按 Ctrl+Alt+T；云服务器 ssh 进去就是终端本体。

Windows 建议 WSL

自带 PowerShell 能用（第 13 页有对照表），但强烈建议装 WSL——在 Windows 里跑一个真正的 Linux。

# 以管理员身份打开 PowerShell，一行命令：
PS> wsl --install
正在安装: Ubuntu ...
# 重启电脑后，开始菜单会出现 "Ubuntu"
# 打开它 = 拥有一个完整的 Linux 终端

为什么值得这一步

装完 WSL，今天后面所有章节（tmux / git / Claude Code / Hermes）的命令你都能原样照抄；全球服务器 95% 以上跑 Linux，学它就是学"通用语"。

WSL = Windows Subsystem for Linux，微软官方功能，Windows 10 1809+ / Windows 11 可用

01 · Terminal · 先看地图再走路

文件系统是一棵树，你永远站在树上某个节点

绝对路径：从根说起

/home/tom/training——从 / 出发的完整地址，在任何位置都有效。

相对路径：从脚下说起

站在 ~ 时，training 就是它；.. = 上一级，. = 当前位置。

波浪号 ~ = 家目录

你的私人领地（/home/你的用户名），迷路了就 cd ~ 回家。

01 · Terminal · 命令组一：走路看路

pwd 我在哪 · ls 有什么 · cd 去哪里

$ pwd                # print working directory 我在哪
/home/tom
$ ls                 # list 列出这里有什么
training  stock-report  notes.txt
$ ls -l              # 加 -l 看详情（大小/时间/权限）
$ ls -a              # 加 -a 连隐藏文件一起看
$ cd training        # change directory 进入文件夹
$ pwd
/home/tom/training
$ cd ..              # 回上一级
$ cd ~               # 一键回家

迷路三连

任何时候不知道自己在哪、有什么、怎么走——pwd → ls → cd，三下就清醒。

Tab 补全 = 半个命令

敲 cd tr 再按 Tab，自动补成 training/。路径几乎从不手打全名。

↑ 箭头 = 历史

按 ↑ 翻出刚才敲过的命令，改两个字再回车。

01 · Terminal · 命令组二：看文件

cat 全倒出来 · less 慢慢翻 · tail -f 盯着看

$ cat notes.txt      # 把整个文件打印出来（适合短文件）
今天要做的事：装 WSL、学 tmux...
$ more notes.txt     # 一页页翻：空格下一页，q 退出
$ less big.log       # more 的升级版：↑↓ 滚动，/ 搜索，q 退出
$ head -5 big.log    # 只看开头 5 行
$ tail -5 big.log    # 只看结尾 5 行
$ tail -f agent.log  # 实时追踪新增内容（Ctrl+C 停）

怎么选

短文件 cat；长文件 less（记住 q 退出）；只关心两头用 head / tail。

tail -f 是盯 Agent 干活的神器

Claude Code、Hermes 跑长任务时都在写日志——tail -f 就是它们的"实时弹幕"。

01 · Terminal · 命令组三：造、搬、删

mkdir 造 · cp 抄 · mv 搬 · rm 删（小心）

$ mkdir demo             # make directory 新建文件夹
$ mkdir -p a/b/c         # -p 一口气建出三层
$ touch todo.txt         # 新建一个空文件
$ cp todo.txt backup.txt # copy 复制
$ cp -r demo demo2       # 复制文件夹要加 -r
$ mv todo.txt done.txt   # move 兼具改名与搬家
$ mv done.txt demo/      # 挪进文件夹
$ rm demo/done.txt       # remove 删文件
$ rm -r demo2            # 删文件夹加 -r

⚠️ rm 没有回收站 刻进 DNA

删了就是删了。删除前先 ls 确认目标；永远、永远不要跑网上抄来的 rm -rf / 之类命令——让 AI 帮你检查一遍再执行。

习惯：重要目录先备份

动手前 cp -r 项目 项目.bak，五秒钟买个后悔药——第三章的 git 是这件事的终极方案。

01 · Terminal · 命令组四：vi 生存指南

vi：不求会用，只求能活着出来

场景：ssh 到服务器改一行配置，那里往往只有 vi。它有"编辑"和"命令"两种模式，新手的一切恐慌都来自不知道自己在哪个模式。

① 打开

vi 文件名——此刻在命令模式，打字没反应是正常的

② 进入编辑

按 i（insert），底部出现 -- INSERT --，现在能正常打字了

③ 退出编辑

按 Esc 回到命令模式（多按几下不吃亏）

④ 保存 / 逃跑

:wq 保存退出；:q! 不保存硬退——逃跑专用

$ vi config.txt
# 按 i → 修改内容 → 按 Esc → 输入 :wq 回车
$ # 改完了，毫发无伤

记不住？正常

把这四招存进手机就够了。日常编辑交给 AI（下一章的 Claude Code 改文件比你快得多），vi 只用来救急。

01 · Terminal · 命令组五：ssh 远程登录

ssh：把千里之外的服务器，变成眼前这个窗口

$ ssh tom@203.0.113.7      # 用户名@服务器地址
Are you sure ...? yes        # 首次连接输 yes
tom@203.0.113.7's password: # 输密码（不显示是正常的）
Welcome to Ubuntu 24.04 LTS
tom@server:~$              # 提示符变了 = 你已在服务器上

# —— 免密登录（一次配置，终身受益）——
$ ssh-keygen                  # 生成密钥对（一路回车）
$ ssh-copy-id tom@203.0.113.7  # 把公钥送上服务器

# —— 传文件 ——
$ scp 本地文件 tom@203.0.113.7:~/   # 上传
$ scp tom@203.0.113.7:~/远端文件 .  # 下载

看提示符认位置

提示符从 本机$ 变成 tom@server:~$，说明你的每条命令都在服务器上执行；exit 回到本机。

手机也能 ssh

装 Termius 等 App，地铁上也能连服务器——这是后面"远程指挥 Agent"的钥匙。

埋个钩子 🪝

ssh 一断线，远端正在跑的程序跟着一起死。怎么办？下一章。

01 · Terminal · ssh 进阶：钥匙与通讯录

公钥是锁，私钥是钥匙，config 是通讯录

$ ssh-keygen -t ed25519       # 生成一对钥匙（可设密码短语）
$ ls ~/.ssh
id_ed25519  id_ed25519.pub  config  known_hosts
#  ↑私钥=钥匙，绝不外传   ↑公钥=锁，随便给
$ cat ~/.ssh/id_ed25519.pub   # 公钥长这样：
ssh-ed25519 AAAAC3Nza... tom@laptop
# ssh-copy-id 干的事 = 把这行追加到服务器的
#   ~/.ssh/authorized_keys（"允许进门的锁清单"）

$ cat ~/.ssh/config           # SSH 的通讯录：
Host work
    HostName 203.0.113.7
    User tom
    IdentityFile ~/.ssh/id_ed25519
$ ssh work                    # 从此四个字母上服务器
$ ssh-add                     # 私钥有口令？钥匙圈替你记住

一对钥匙的分工

公钥（.pub）＝锁：装到服务器的 authorized_keys 门上；私钥＝钥匙：只留在你电脑里开门。给别人公钥没风险，私钥离开电脑就等于把家门钥匙发到了群里。

~/.ssh/config ＝ 通讯录

给每台服务器起外号，地址、用户名、用哪把钥匙一次写清——之后 ssh work、scp 文件 work:~/ 全都认它。管两台以上服务器的必备品。

ssh-add ＝ 钥匙圈

私钥设了密码短语更安全，但每次连接都要输？ssh-add 把私钥交给 ssh-agent 保管，开机后只输一次；ssh-add -l 看圈上挂了哪些钥匙。

反向理解 authorized_keys：往里加一行公钥 ＝ 给对方配一把你家钥匙。所以第 30 页让 Claude Code 帮忙改配置时，动这个文件的操作要亲自审批。
01 · Terminal · 命令组六：求救三招

记不住命令不丢人，会求救才是本事

第一招：--help

几乎所有命令都认：ls --help、tmux --help——一屏速览用法。

第二招：man

man ls 打开完整说明书（manual），q 退出。经典但啰嗦。

第三招：问 AI 2026 正确姿势

把报错原文整段贴给 AI；或直接说目标："帮我把这个文件夹里所有 mp4 转成 mp3"——命令让它写，你负责看懂再回车。

$ claude -p "用一条命令统计当前目录下有多少个 txt 文件"
ls *.txt | wc -l
新的分工：你描述目标 + 审查命令，AI 负责记忆语法。唯一红线：不理解的删除/系统级命令，先让 AI 解释每个参数再执行。

02

tmux：关掉电脑，工作还在跑

上一章埋的钩子：ssh 一断，远端程序就死。tmux 让你的会话常驻服务器——断线只是"暂时不看"，不是"全部重来"。

02 · tmux · 为什么与是什么

把会话寄存在服务器上的"保险箱"

没有 tmux 的惨剧

你 ssh 到服务器让 Agent 跑一个 3 小时的任务 → 进电梯，Wi-Fi 断了 3 秒 → 任务当场死亡，从头再来。

有 tmux 的日常

任务跑在 tmux 会话里 → 断线、关电脑、回家睡觉都无所谓 → 第二天 tmux attach，现场原样还在。

三层结构

Session（会话）＞ Window（窗口，像浏览器标签页）＞ Pane（窗格，一屏切几块）。新手记住 Session 就够。

02 · tmux · 四个保命操作

只需四招：开、挂、列、接

$ sudo apt install tmux   # 装（Mac: brew install tmux）
$ tmux new -s ai          # ① 开一个叫 ai 的会话
# ……在里面启动你的长任务……
# ② 挂起：按 Ctrl+b 松手，再按 d（detach）
[detached (from session ai)]
$ tmux ls                 # ③ 列出所有会话
ai: 1 windows (created Fri Jul  4 10:02)
$ tmux attach -t ai       # ④ 接回现场，原样如初

# 加分项：分屏
# Ctrl+b %  左右分屏 | Ctrl+b "  上下分屏
# Ctrl+b 方向键  在窗格间移动

指挥键 Ctrl+b

tmux 所有快捷键都是"先按 Ctrl+b 松手，再按一个键"。今天只需记住 d（挂起）。

挂起 ≠ 关闭

detach 后程序继续跑；在会话里敲 exit 才是真正关闭。

命名好习惯

会话按用途取名：tmux new -s claude、-s hermes——回来时一眼认出。

02 · tmux · 本章点题

tmux 是 Agent 的宿舍：住进去，就不下班

白天 · 办公室

ssh 上服务器，tmux new -s cc，让 Claude Code 开始重构项目。

傍晚 · 地铁上

Ctrl+b d 挂起、合上电脑。任务在服务器上继续跑。手机想看进度？Termius ssh 上去 tmux attach。

深夜 · 家里

打开家里电脑 tmux attach -t cc——下午的现场一秒回来，Agent 已经干完了。

一句话：ssh 给你"到场"的能力，tmux 给你"离场"的自由。第四章 Claude Code 的官方 Remote Control 会把这套体验再升一级。
03

Git / GitHub：代码的时光机

rm 没有回收站，但 git 有——而且是全宇宙最强的那种。它还是全球程序员的协作协议，和你未来最好看的一份简历。

03 · Git / GitHub · 为什么人人都该会

四个理由，一个比一个现实

① 时光机 后悔药

每次 commit（提交）都是一个存档点。改崩了？一条命令回到任意存档。写论文都想要的功能。

② 协作协议 全球通用

几万人共写一个项目而不打架，靠的就是 git 的分支与合并。全球程序员的"普通话"。

③ AI 时代的安全网 本课重点

你敢让 Claude Code 大刀阔斧改代码，底气就是 git：AI 改的每一步都有记录，不满意一键回滚。没有 git 的项目，AI 也不敢放开手脚。

④ GitHub = 新简历 对学生最实际

雇主越来越先看 GitHub 再看简历。你做过的练习、写过的项目，push 上去就是可验证的作品集。

概念一句话：git 是装在你电脑上的版本管理工具；GitHub 是把仓库托管到云端的网站（还有 Gitee、GitLab 等同类）。
03 · Git / GitHub · 心智模型

三步走：改 → 存档 → 上云

add = 把改动放进"待存档"篮子

commit = 咔嚓一声存档，附一句说明

push / pull = 和云端同步：推上去 / 拉下来

03 · Git / GitHub · 日常五连

一个仓库的一天

$ git init                 # 把当前文件夹变成 git 仓库（一次性）
$ git status               # 口头禅：现在什么状态？
modified: index.html
$ git add .                # 把所有改动放进暂存区
$ git commit -m "完成报名表单"  # 存档 + 一句人话说明
$ git log --oneline        # 看存档历史
a3f9c21 完成报名表单
9b2d1e0 初始版本
$ git diff                 # 看看这次到底改了什么
$ git checkout a3f9c21 -- index.html  # 后悔药：取回旧版

口头禅：git status

任何时候心里没底，先 status——它会告诉你现在在哪、改了啥、下一步能干嘛。

commit 说明写"为什么"

"改了 index.html" ❌（git 自己知道）；"修复手机端按钮点不到" ✅（未来的你会感谢）。

多小算一次提交？

完成一个说得清的小目标就存档一次。小步快存，时光机才好用。

03 · Git / GitHub · 上云三件事

GitHub：备份、展示、协作，一站全包

# ① 克隆别人的项目（比如龙虾）
$ git clone https://github.com/openclaw/openclaw.git
Cloning into 'openclaw'...

# ② 把自己的仓库推上云（在 GitHub 网页先建空仓库）
$ git remote add origin https://github.com/你/my-site.git
$ git push -u origin main   # 首次推送；之后只需 git push

# ③ 换台电脑继续干 / 同步队友的改动
$ git pull

注册即拥有

github.com 免费账号 = 无限公开仓库。用户名认真取，它会出现在你的简历里。

README.md 是门面

仓库首页自动展示 README：项目是什么、怎么跑、效果截图。写好 README 的仓库才算作品。

星标 = 收藏 + 点赞

上一讲里龙虾 38 万星、Hermes 21 万星，就是这个星。给你用过的好项目点个星，也是圈内社交。

进阶彩蛋 🥚：GitHub 官方命令行工具 gh——网页上的操作全能在终端完成。gh auth login 登录一次，之后 gh repo create my-site --public --source . --push 一条命令搞定"建仓库 + 关联 + 推送"三件事，发 PR 用 gh pr create。等左边三件事玩顺了，再来解锁它。
03 · Git / GitHub · 本章点题

AI 时代，git 反而更重要了

AI 的安全绳

让 Claude Code 放手重构的前提是"随时可回滚"。先 commit，再让 AI 动手——这是 AI 协作第一守则。

AI 的原生语言

Claude Code 本来就用 git 干活：自动建分支、写提交说明、发 PR。你懂 git，才看得懂它在干什么、拦得住它跑偏。

你的复利资产

从今天的练习开始，每个作品都 push 上 GitHub。一年后你会拥有别人补不出来的东西：带时间戳的成长记录。

git 学到这个深度就够用了——分支、合并、PR 这些进阶概念，等你需要时让 AI 现场教你，十分钟的事。
04

Claude Code：终端里的编程搭档

前三章的所有技能在这里合流：你在终端里指挥它，它用命令和 git 干活，tmux 让它跑个通宵——而且现在，手机就能接管。

04 · Claude Code · 安装与启动

一行命令安装，一个 claude 启动

# macOS / Linux / WSL：
$ curl -fsSL https://claude.ai/install.sh | bash

# Windows PowerShell（不用 WSL 也能装）：
PS> irm https://claude.ai/install.ps1 | iex

# 启动：进入你的项目文件夹，然后——
$ cd my-project
$ claude
按提示在浏览器完成登录 ✓
> 帮我看看这个项目是干什么的▌

登录方式

Claude Pro / Max 订阅账号直接 OAuth 登录（推荐）；或用 API key。/login 可随时重新登录。

三种打开方式

claude 交互会话；claude -c 继续上次会话；claude -p "问题" 无头模式——问完即走，适合写进脚本。

第一件事

在项目里输入 /init，让它生成 CLAUDE.md（后面第 29 页专讲）。

安装命令与支持平台来自官方文档 code.claude.com/docs（2026-07 核实）：macOS 13+ / Ubuntu 20.04+ / Windows 10 1809+

04 · Claude Code · 日常核心操作

六个动作，覆盖每天 90% 的使用

动作怎么做
提需求直接打字说人话，越具体越好
打断它Esc ——说错了、跑偏了随时喊停
翻历史↑ ↓ 箭头
切权限模式Shift+Tab 循环：逐条确认 → 自动接受编辑 → 计划模式
直跑 shell!ls ——叹号开头，绕过 AI 直接执行
喂图片截图直接粘贴进对话（报错截图、设计稿都行）

权限模式是安全带

默认每个危险操作都先问你；熟练后用"自动接受编辑"；让它先出方案再动手，用计划模式（plan）。

好需求的三要素

目标 + 约束 + 验收标准："给报名页加一个导出 CSV 按钮，不改现有样式，导出后能用 Excel 打开。"

04 · Claude Code · 高频斜杠命令

输入 / 呼出命令面板，这 8 个最常用

/init扫描项目，生成 CLAUDE.md 项目说明
/help看所有可用命令
/clear清空对话重新开始（项目记忆保留）
/model切换模型（快活儿用小模型，硬活儿用大模型）
/plan进入计划模式：先出方案，批准了才动手
/resume恢复之前的会话，接着上次干
/memory查看 / 编辑它记住的东西
/permissions管理它能自动做什么、必须问什么

省钱小知识

对话越长越贵（每次都要"重读"上下文）。一个任务干完就 /clear，换任务换会话。

会话不怕丢

意外关掉终端？claude -c 继续最近会话，claude --resume 挑历史会话——像 tmux 一样有"存档"。

命令清单来自官方文档 code.claude.com/docs/en/commands（2026-07 核实）

04 · Claude Code · 项目记忆

CLAUDE.md：写给 AI 的"项目说明书"

$ cat CLAUDE.md
# training — AI 培训幻灯片项目

## 设计系统（改动时必须遵守）
- 深色科技风，色板 token 在 :root
- 美国=青蓝 #56C8F5，中国=暖橙 #F5A340
- 字号必须随屏幕缩放，禁止固定 px 上限
- 所有数据必须联网核实并标注来源+日期 ...

它是什么

放在项目根目录的 Markdown 文件，每次启动会话自动读取。写什么：构建命令、代码规范、设计规则、踩过的坑。

真实案例：你们看过的那份 PPT

左边就是第一讲幻灯片项目的真实 CLAUDE.md 节选——因为有它，任何一次新会话都能接着改，不会忘记"字号要随屏幕缩放"这种教训。

经验

短小精悍（200 行以内最有效）；每次 AI 犯了同样的错，就往里加一条规矩——它是你和 AI 的共同备忘录。

04 · Claude Code · Remote Control（官方远程遥控）

电脑上开跑，手机上接管

# 方式一：给现有会话开启远程（在会话里输入）
> /remote-control 我的项目
✓ 已开启 · 出示二维码 · claude.ai/code 可见

# 方式二：启动时就带远程
$ claude --remote-control "重构任务"

# 方式三：纯服务器模式（多设备连接）

手机怎么接

Claude 官方 App（iOS/Android）扫码即接管：看进度、发指令、批准操作；或任何浏览器打开会话 URL。

推送通知

/config 里开启 "Push when actions required"——AI 需要你拍板时，手机弹通知，像外卖到了一样。

唯一前提

本地进程必须活着（关终端 = 会话结束）——所以它和 tmux 是天生一对，下一页组合拳。

功能与命令来自官方文档 code.claude.com/docs/en/remote-control（2026-07 核实）；另有 Telegram/Discord 消息触发（Channels）、定时任务（Routines）等入口

04 · Claude Code · 终极形态：三章合体

云服务器 + tmux + Remote Control = 永不下班的工程师

Hermes Agent：部署你的常驻管家

Claude Code 是"编程搭档"，Hermes 是"生活管家"——装在服务器上 7×24 待命，微信里发句话就干活，而且越用越懂你。

05 · Hermes Agent · 安装部署（v0.18，2026-07）

一条命令，把 Hermes 请进服务器

# 先给它安排宿舍（第 2 章学的）：
$ ssh tom@203.0.113.7
$ tmux new -s hermes

# 官方一键安装（Linux / macOS / WSL2 通用）：
$ curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
✓ Python · Node.js · ffmpeg 等依赖自动装好
✓ 全局命令 hermes 已可用
$ hermes doctor      # 体检：环境有没有装好

脚本包办一切

依赖（Python、Node、ffmpeg…）全部自动安装，官方口径"两分钟跑起来"。无头服务器可加 --skip-browser。

平台支持

macOS / Windows / Linux / WSL2 / Docker 都是一级支持；云服务器（Linux）正是官方推荐姿势。Windows 原生用 iex (irm …/install.ps1)。

为什么装在服务器

管家要 7×24 值班——跑在 tmux 里的云服务器上，你的电脑关不关机都不影响它上班。

05 · Hermes Agent · 首次配置与接入

三步：接上大脑 → 启动网关 → 连上微信

$ hermes setup            # ① 交互式向导：跟着答题就配好
$ hermes model            # 选"大脑"：支持 30+ 家模型
  Anthropic Claude / OpenAI / DeepSeek / Qwen ...

$ hermes gateway setup    # ② 配置要接入的聊天平台
$ hermes gateway start    # 启动常驻网关（在 tmux 里！）

# ③ 手机上给你的 bot 发第一条消息 → 把自己的
#    用户 ID 加进允许名单 → 重启网关，开始使唤它

中文平台原生支持

微信、企业微信、飞书、钉钉、QQ，加上 Telegram / Discord / Slack / 邮件——共 20+ 平台，一个网关全接住。

文件都在哪

配置 ~/.hermes/config.yaml；API key 放 ~/.hermes/.env（记得 chmod 600）。

⚠️ 安全三条，一条别破

① 永远不开"允许所有人"（ALLOW_ALL_USERS）；② 危险操作保持默认的人工审批，别图省事开 /yolo；③ 别用 root 跑网关。

来源：官方 Quickstart / CLI Reference / Weixin 接入文档 / Security 文档（2026-07-04 核实）

05 · Hermes Agent · 技能与记忆

看着它一天天变聪明

$ hermes --tui            # 推荐的终端界面
> /skills                 # 看它已经学会的技能
  ├─ 整理下载文件夹
  ├─ 每晚备份 stock-report

$ ls ~/.hermes/skills/    # 技能 = 一个个文件夹，看得见摸得着
$ cat ~/.hermes/memories/MEMORY.md  # 它对你的记忆，也是明文 md

"自我进化"落地的样子

你让它做成一件复杂的事，它自动写成技能文档存进 skills/——下次同类任务直接调用，还会越用越顺。这就是第一讲说的"与你共同成长"。

记忆全透明

它记住的一切都在 MEMORY.md / USER.md 两个明文文件里，可以直接编辑或删除——你的管家，你做主。

技能可以"进货"

hermes skills install 安装社区技能（兼容开放技能标准）——站在全球用户的肩膀上养管家。

来源：官方 CLI Reference / Persistent Memory / Skills 文档（2026-07-04 核实）
